# Phase 20: Add candlestick PNG plots + stdout ASCII support
**Priority:** P1 / base_plan P11  
**Module:** `src/plot.rs`, `src/pipeline.rs`, `src/main.rs`

## Goal
Add candlestick chart PNG output and optional ASCII terminal output for alpha visual polish.

## Tasks
1. In `src/plot.rs`, add `plot_candlestick(df, open_col, high_col, low_col, close_col, title, output_path)`:
   - Use plotters `Candlestick` series. Rise = GREEN, Fall = RED.
   - X axis = `0..n_rows` with `date` x-label formatter (mirror `plot_line_chart`).
   - Return `Err` on empty df or missing OHLC columns.
2. In `src/plot.rs` (or new `src/ui_text.rs`), add `print_candlestick_stdout(df, col, max_width)`:
   - Downsample to terminal width (default 80).
   - Block glyphs + ANSI color (green rise / red fall).
   - Guard single-row / all-flat.
3. In `src/pipeline.rs`, add `candle_stdout: bool` to `PipelineConfig`.
   - In the plotting block, per `currency_cols`, write `{run_dir}/{col}_candlestick.png` gated by `config.plots && !light`.
   - If `config.candle_stdout`, print ASCII candles to stdout per `currency_cols` (independent of `light`/`plots`).
4. In `src/main.rs`, add `--candle-stdout` to `RunPipeline` (and optionally `Ohlcv` subcommand); pass through.
5. Add tests:
   - `plot_candlestick` → valid PNG signature `[137,80,78,71,13,10,26,10]`.
   - `print_candlestick_stdout` → non-empty, contains block glyphs, no panic on single-row/all-flat.
   - Missing OHLC col → `Err` gracefully.

## Validation
- `cargo test -p cdg plot::` passes (tests render to `std::env::temp_dir()`).
- Manual: `cdg collect --coin btc,eth` → `{col}_candlestick.png` in run dir.
- Manual: `cdg collect --coin btc --candle-stdout` → ASCII candles printed.

## Why Included
Visual artifacts are the first thing alpha users see. Candlestick PNG + ASCII is an impressive, low-risk addition that sets CDG apart from CLI-only tools.
