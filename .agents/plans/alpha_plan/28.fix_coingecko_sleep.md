# Phase 28: Remove hardcoded 3s sleep on every cache miss
**Priority:** P2 — Performance / correctness
**Module:** `src/api/coingecko.rs`
**Source:** `code_review.md` #4

## Goal
Remove the unconditional `tokio::time::sleep(Duration::from_millis(3000))` that fires on every cache miss regardless of context.

## Tasks
1. Delete the blanket 3s `sleep` in the `get_coingecko_*` request path (reported at `coingecko.rs:~74`).
2. Rely on the exponential-backoff retry logic from Phase 12 which only delays on 429 / 5xx / network errors.
3. Ensure no other unconditional `sleep` remains in the fetch path.
4. Add a manual timing check: a single-coin cache miss completes without a forced 3s delay.

## Validation
- `cargo test -p cdg` passes.
- Manual: `cdg collect --coin btc` cold run no longer adds ~3s × request-count of forced latency (5 benchmarks + 1 coin × 2 endpoints = ~12 requests → was ≥36s of forced sleep).

## Why Included
`code_review.md` #4 flags this as a Warning: the sleep fires on every cache miss even for single-coin queries, adding ≥36s latency per run. Only needed to avoid 429, which Phase 12 backoff already handles.
