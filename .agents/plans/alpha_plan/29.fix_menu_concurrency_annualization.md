# Phase 29: Thread concurrency + annualization into interactive menu
**Priority:** P2 — Consistency
**Module:** `src/main.rs`, `src/ui.rs`
**Source:** `code_review.md` #6

## Goal
Make the interactive menu honor the same `--concurrency` and `--annualization-factor` options the CLI exposes, instead of hardcoding them.

## Tasks
1. In the interactive menu flow (`ui.rs` → `main.rs` call site reported at `main.rs:~880` as `run_pipeline_flow(..., 3, None)`):
    - Prompt (or default) for `concurrency` and `annualization_factor` and pass them through, matching the CLI `RunPipeline` behavior.
2. Remove the hardcoded `3` concurrency and `None` annualization; derive from menu config or documented defaults.
3. Add a test or manual check that menu-selected concurrency/annualization reach `run_pipeline_flow` (e.g., via a config struct assertion or logged values).

## Validation
- `cargo test -p cdg` passes.
- Manual: launch interactive menu, set concurrency=5 and annualization=252 → run honors both (observable via logs / report Sharpe factor).

## Why Included
`code_review.md` #6 is a Warning: the interactive menu silently ignores two CLI-exposed options, producing inconsistent results between CLI and TUI. Low risk, high perceived-correctness value.
