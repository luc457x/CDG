# Phase 18: Add weekend fill semantics regression tests
**Priority:** P2 — Locks alpha-critical fill behavior  
**Module:** `src/analysis.rs` test module (base_plan P12)

## Goal
Regresssion anchor ensuring `_volume` columns receive Zero-fill on weekend gaps, protecting Backlog P9/B1.

## Tasks
Add the following tests to `analysis.rs` `#[cfg(test)]`:

1. **T12a** — Crypto base (7d) LEFT JOINed with stock-like series (Friday close=100, Friday volume=1000, Saturday/Sunday all null). Test both:
   - `drop_weekends=false` → Saturday/Sunday rows retained; price forward-filled to 100, **volume = 0**.
   - `drop_weekends=true` → weekend rows dropped; validate that the forward-fill happened **before** the drop (i.e., Friday close carried forward logically).
2. **T12b** — Standalone `{ticker}_volume` column with weekend nulls → asserts the Zero branch is taken (not forward-fill), anchoring the `P9/B1` dependency where stock OHLCV volume columns MUST use `_volume` suffix.

## Validation
- `cargo test -p cdg analysis::` passes.
- No source change unless test fails.

## Why Included
Zero-fill for volume is a subtle semantic choice. Alpha users mixing crypto and stock data will encounter this. A regression test costs nothing and prevents future "fixes" from breaking it.
