# Phase 07: Replace Ctrl+C std::process::exit with CancellationToken
**Priority:** P0 — Alpha Blocking  
**Module:** `src/pipeline.rs` (Phase 2 / alpha 1.3)

## Goal
Graceful shutdown on SIGTERM / Ctrl+C. Stop abandoning `JoinSet` tasks, half-written files, and SQLite WAL handles.

## Tasks
1. Add `tokio_util::sync::CancellationToken` to the project (already a transitive dep? check `Cargo.toml`; if missing, add `tokio-util`).
2. In `run_pipeline_flow` (and `run_standalone_backtest` if it spawns tasks):
   - Create a `CancellationToken`.
   - Pass `token.clone()` into every spawned fetch task.
   - Each task checks `token.is_cancelled()` in its loop and returns early if true.
   - On Ctrl+C, call `token.cancel()`, then `join_set.shutdown().await` to drain remaining tasks.
3. Remove the `std::process::exit(0)` handler entirely.
4. In `main.rs`, after `run_pipeline_flow` / `run_standalone_backtest` returns:
   - If the error is cancellation, exit with a clean non-zero code or `Ok(())` if user-initiated. Do not use `std::process::exit`.
5. Ensure file handles (CSV/Parquet writers) and SQLite pools are dropped naturally by Rust ownership before `main` returns.

## Validation
- `cargo run -- run-pipeline -c bitcoin -d 30` mid-fetch, Ctrl+C → no half-written files in output dir, `.db` not locked, process exits cleanly.
- `cargo test` passes.

## Why This Blocks Alpha
Cloud Run / containers send SIGTERM regularly. A tool that leaves half-written CSVs and locked DB files is not alpha-ready.
