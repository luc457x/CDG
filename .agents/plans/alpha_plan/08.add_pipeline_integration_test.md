# Phase 08: Add mocked HTTP integration test for run_pipeline_flow
**Priority:** P0 — Alpha Blocking  
**Module:** `tests/pipeline_tests.rs` (Phase 3 / alpha 2.2)

## Goal
End-to-end regression test that exercises `run_pipeline_flow` against fake HTTP responses so cache/api/join changes cannot silently break the flow.

## Tasks
1. Add `wiremock` (or `mockito`) as a dev-dependency in `Cargo.toml`.
2. Spin up a mock HTTP server that serves canned JSON for:
   - CoinGecko `/coins/{id}/market_chart/range`
   - CoinGecko `/coins/{id}/ohlc`
   - Yahoo Finance `/v8/finance/chart/{ticker}`
3. Call `run_pipeline_flow` with the mock server URL as the API base.
4. Assert that the run directory contains at minimum:
   - `data.csv`
   - `data.parquet`
   - `portfolio_weights.csv`
   - One PNG file
5. Add negative scenario tests:
   - One coin returns 404 → assert graceful skip + warning.
   - `^TNX` missing from Yahoo → assert benchmark degradation (no panic, no missing benchmark panic).
   - Cache hit on second run → faster execution, assert cache hit logs.

## Validation
- `cargo test --test pipeline_tests` integration case passes.
- Existing unit tests still pass.

## Why This Blocks Alpha
A change in `cache.rs`, `api/coingecko.rs`, or `pipeline.rs` join logic could break the entire flow without any test catching it. This is the single most important regression test before alpha.
