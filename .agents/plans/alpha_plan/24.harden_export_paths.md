# Phase 24: Harden export.rs paths
**Priority:** P2 — Medium severity hygiene  
**Module:** `src/export.rs`

## Goal
Remove unnecessary `&mut` binding, prevent path-injection, and fix test file pollution.

## Tasks
1. Change signatures from `&mut DataFrame` to `&DataFrame` in both `export_csv` and `export_parquet` (polars writers take `&DataFrame`).
2. Add `.with_context(|| format!("creating parent dir for {}", path))` to `create_dir_all` calls.
3. Call `utils::validate_safe_path(path)` before any file operation.
4. Replace hardcoded `tests/temp_test.csv` / `tests/temp_test.parquet` in tests with `tempfile::NamedTempFile` (or `tempfile::TempDir`).
5. Optional: atomic write — write to `path.tmp()` then `std::fs::rename` to final path. For alpha, the tempfile in tests is sufficient; atomic write in production code can be backlog.

## Validation
- `cargo test -p cdg export::` passes.
- Run tests twice in a row → no leftover `.csv` / `.parquet` in `tests/`.
- `cargo clippy` does not warn about unused mut.

## Why Included
Test pollution leaves artifacts in tracked dirs. `&mut DataFrame` increases borrow-checker friction for no benefit. Both are quick wins.
