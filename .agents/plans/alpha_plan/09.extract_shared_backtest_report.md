# Phase 09: Extract shared backtest report + treasury benchmark logic
**Priority:** P1 — Must fix before wider alpha distribution  
**Module:** `src/backtest.rs`, `src/pipeline.rs`

## Goal
Remove ~200 lines of copy-pasted backtest/report logic from `pipeline.rs` and centralize it.

## Tasks
1. In `backtest.rs`, add:
   - `generate_backtest_report(metrics: &BacktestMetrics, run_dir: &Path, asset: &str) -> Result<()>` that writes CSV + JSON + formatted table.
   - `append_treasury_benchmark(df: &mut DataFrame, cols: &[String], ann_factor: f64, metrics: &mut BacktestMetrics) -> Result<()>` for `^TNX` logic.
2. Replace the duplicated blocks in `run_pipeline_flow` and `run_standalone_backtest` with calls to these functions.
3. Fix ancillary issues in `pipeline.rs` report path:
   - `HashMap` key collision warning (line ~714): surface to user instead of silently overwriting.
   - JSON write failure (line ~767): return `Err` or at least warn; do not print unconditional success if JSON failed.
   - Final success message (line ~767): reflect actual status (plots skipped? optimization skipped?) instead of unconditional "Pipeline completed successfully".
4. Reduce `pipeline.rs` to <400 LOC in the orchestration functions.

## Validation
- `cargo build` clean.
- Both `run_pipeline_flow` and `run_standalone_backtest` produce identical report bytes for the same input (golden compare).
- `cargo test -p cdg` passes.

## Why Included
God-functions in `pipeline.rs` make every alpha bug fix twice as expensive. Centralizing report generation is a strong maintainability win with low risk.
