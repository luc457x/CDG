# Phase 01: Fix analysis.rs parser bugs
**Priority:** P0 — Alpha Blocking  
**Module:** `src/analysis.rs` (Phase 4a findings)

## Goal
Fix data corruption in CoinGecko and Yahoo parsers. These bugs silently corrupt the price/OHLCV series that every downstream indicator, backtest, and Monte Carlo run depends on.

## Tasks
1. **Fix `parse_coingecko_market_chart` panic** (line ~27): Replace `total_volumes[i]` with `total_volumes.get(i).map(|(_, v)| v).unwrap_or(0.0)` to prevent OOB panic when `prices.len() > volumes.len()`.
2. **Fix `parse_coingecko_market_chart` aggregation**: If multiple prices land on the same date, use the **last** close of the day, not `mean`. Only `high` stays `max`, `low` stays `min`.
3. **Fix `parse_coingecko_ohlc` aggregation** (lines ~91-94): `open` must be the **first** row of the day, `close` must be the **last** row of the day. Replace `.mean()` with `.first()` / `.last()`.
4. **Fix `parse_coingecko_ohlc` malformed rows** (line ~64): Replace silent drop of `item.len() < 5` with `eprintln!` warning + skip, or return `Err`. At minimum, log the skipped row.
5. **Fix `parse_yahoo_json` null continuity** (lines ~274-282): When `price_opt` is `None`, do not drop the row. Forward-fill the previous price or insert `None` and let `align_datasets` handle it. The exact strategy: keep the date, fill the price with `None`, then downstream `align_datasets` forward-fills.

## Validation
- `cargo test -p cdg` — all existing tests pass.
- Add/verify tests in `analysis.rs`:
  - Market chart with prices > volumes → no panic, missing volumes → `0.0`.
  - Multi-price day → last close wins.
  - OHLC malformed row → logged warning, not silent drop.
  - Yahoo null price → date continuity preserved in output DataFrame.

## Why This Blocks Alpha
A single missing OHLC bar or malformed CoinGecko response currently produces garbage ATR/Stoch/ADX values or panics. Alpha users will hit this on day one.
