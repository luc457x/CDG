# Phase 17: Add weekend alignment tests
**Priority:** P2 — Locks alpha-critical weekend gap behavior  
**Module:** `src/analysis.rs` test module (base_plan P3)

## Goal
Lock in the correct `align_datasets` + `drop_weekends` behavior with explicit test vectors. No source change unless a test fails.

## Tasks
Add the following tests to `analysis.rs` `#[cfg(test)]`:

1. **T1** — DataFrame with Sat+Sun rows + weekdays, `drop_weekends=true` → weekend rows absent, weekday rows retain forward-filled values.
2. **T2** — Same data, `drop_weekends=false` → all rows present, weekend nulls forward-filled (no NaN).
3. **T3** — `{ticker}_volume` column with weekend nulls → filled with **Zero**, not forward-filled.
4. **T4** — crypto(7d) LEFT JOINed with stock-like series (5d weekdays), `drop_weekends=true` → only weekdays remain, stock nulls forward-filled from Friday, no gaps.
5. **T5** — Unparseable date string (e.g., `"not-a-date"`) → mask keeps row (`true`) — documents current behavior.
6. **T6** — `parse_coingecko_market_chart` synthetic 7-day JSON including a weekend → confirms weekend rows exist pre-align.

## Validation
- `cargo test -p cdg analysis::` passes.
- If any test fails, fix the source in the same phase before marking complete.

## Why Included
Weekend alignment is a correctness contract for crypto + stock joins. Alpha users will combine these datasets. Locking behavior with tests prevents regressions.
