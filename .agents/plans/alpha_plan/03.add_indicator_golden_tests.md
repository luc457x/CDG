# Phase 03: Add golden-value unit tests for all technical indicators
**Priority:** P0 — Alpha Blocking  
**Module:** `src/analysis.rs`

## Goal
Validate numerical correctness of all 8 indicator implementations (`sma`, `ema`, `rsi`, `macd`, `bollinger`, `atr`, `stochastic`, `adx`, `obv`) against manually computed reference vectors.

## Tasks
1. Write test vectors for each indicator function. Each gets **3 cases** minimum:
   - Uptrend (e.g., 20 close prices rising 100→200)
   - Downtrend (e.g., 20 close prices falling 200→100)
   - Flat / sideways (e.g., 20 close prices at 150 ± 2)
2. For each case, compute the expected values by hand or with `pandas_ta` / `ta` in Python, then assert exact float equality (or `assert!((actual - expected).abs() < 1e-6)`).
3. Add edge-case tests for each indicator:
   - `period > data.len()` → all `None`
   - Single-bar DataFrame → all `None`
   - All identical closes (zero variance) → defined behavior (e.g., RSI = 50.0 or `None`)
   - NaN propagation: input contains `NaN` → output contains `NaN` (do not silently produce numbers)
4. Ensure the existing `test_returns_and_indicators_computation` assertion is upgraded from column-existence to also checking:
   - RSI bounded `[0.0, 100.0]` where not `None`
   - Bollinger upper > lower where not `None`
   - MACD histogram = line - signal where not `None`
   - OBV is monotonic on flat price (changes only with volume)

## Validation
- Minimum 15–20 new `#[test]` functions in `analysis.rs`.
- `cargo test -p cdg analysis::` passes.
- No indicator is released without at least one golden-value test.

## Why This Blocks Alpha
Alpha users compare CDG outputs to TradingView/Excel. If RSI has an off-by-one or wrong smoothing, every backtest result is silently corrupted. These are hand-rolled implementations with no external validator.
