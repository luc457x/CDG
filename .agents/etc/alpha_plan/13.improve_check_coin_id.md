# Phase 13: Improve check_coin_id return type + coin list lowercasing
**Priority:** P1 — Must fix before wider alpha distribution  
**Module:** `src/api/coingecko.rs` (line_by_line 8a)

## Goal
Make `check_coin_id` robust against ambiguous matches and eliminate per-iteration allocations.

## Tasks
1. Change return type from `Option<String>` (or similar) to a typed enum:
   - `CoinResolution::Exact(String)` — single unambiguous match.
   - `CoinResolution::Ambiguous(Vec<String>)` — multiple matches, surface to user.
   - `CoinResolution::NotFound` — no match.
2. Lowercase the entire `/coins/list` response once after fetching/caching. Do not lowercase on every iteration.
3. Update callers in `main.rs` and `ui.rs` to match the new enum and surface ambiguity.

## Validation
- Unit tests:
  - `test_check_coin_exact` → `Exact("bitcoin")`.
  - `test_check_coin_ambiguous` → `Ambiguous(["bitcoin-cash", "bitcoin"])`.
  - `test_check_coin_not_found` → `NotFound`.
- `cargo test -p cdg` passes.

## Why Included
Interactive menu and `check-coin` CLI currently fail ambiguously or silently. Alpha users need clear feedback.
