# Phase 02: Fix OHLC null fill corruption in compute_returns_and_indicators
**Priority:** P0 — Alpha Blocking  
**Module:** `src/analysis.rs` (Phase 4d findings)

## Goal
Stop replacing missing `high`, `low`, and `volume` values with `0.0` inside `compute_returns_and_indicators`. `0.0` creates garbage ATR, Stochastic, ADX, and OBV values that silently propagate to backtest results.

## Tasks
1. In `compute_returns_and_indicators`, change:
   - `highs_raw[i].unwrap_or(0.0)` → `highs_raw[i].unwrap_or(0.0)` becomes `highs_raw[i].unwrap_or(None).flatten()` or construct `Vec<Option<f64>>` and propagate `None` into ATR/Stoch/ADX/OBV computations.
   - Same for `lows_raw` and `volumes_raw`.
2. Propagate `None` through `calculate_atr`, `calculate_stochastic`, `calculate_adx`, and `calculate_obv` so that any bar where `high`, `low`, or `volume` is missing emits `None` for the dependent indicator at that index.
3. Add `data_quality_warnings: Vec<String>` to the function return (or a `Vec<String>` push into a logger) listing dates where OHLC was missing and was therefore gated. Example: `"bitcoin_usd: 2026-06-15 high was null — ATR/ADX/Stoch set to None"`.
4. Update `prep_ml` (Phase 4d): guard against `inf`/`nan` inputs before min/max/std calculation. Document that constant-variance fallback `std = 1.0` is intentional (avoids division by zero).

## Validation
- Test: DataFrame with one missing `high` at index 5 → `atr_14[5]` is `None`, not `0.0` or infinity.
- Test: missing `volume` at index 3 → `obv[3]` is `None`, not unchanged.
- Test: `data_quality_warnings` contains the date string of the missing OHLC bar.
- `cargo test -p cdg analysis::` passes.

## Why This Blocks Alpha
Alpha users will run 90-day pipelines on real exchange data. Real data has gaps. A single null OHLC bar currently corrupts all derived indicators.
