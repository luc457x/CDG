# Phase 06: Add deterministic equity-curve and portfolio rebalancing tests
**Priority:** P0 — Alpha Blocking  
**Module:** `src/backtest.rs`

## Goal
Validate that signal generation, fee application, and portfolio rebalancing produce exact, deterministic results on known inputs.

## Tasks
1. **Strategy signal tests** — feed deterministic DataFrames into `run_backtest_for_asset` and assert exact equity curve values at each bar:
   - RSI strategy: prices that trigger RSI < 30 → assert exact equity after buy/hold/sell.
   - MACD strategy: prices that produce a known cross → assert exact equity before/after cross.
   - Bollinger strategy: prices that touch upper/lower band → assert exact equity.
   - Custom JSON strategy: load a 2-rule JSON config, evaluate against known DataFrame, assert exact equity.
2. **Edge-case strategy tests**:
   - Price gap up/down between bars.
   - Zero volume day.
   - All `None` indicator columns → strategy falls back to neutral; equity stays flat minus fees.
   - Single-bar DataFrame → returns equity unchanged (no fee).
3. **Portfolio rebalancing tests** — call `backtest_portfolio` with deterministic asset returns:
   - Weekly rebalance: assert exact fee deductions and weight drift between rebalances.
   - Monthly rebalance: same.
   - Multi-asset with diverging returns: assert weights renormalize correctly after fee.

## Validation
- Minimum 10–15 new `#[test]` functions in `backtest.rs`.
- `cargo test -p cdg backtest::` passes.
- Every new test asserts exact equity values (not just "no error").

## Why This Blocks Alpha
Strategy backtesting is the core value proposition. If the signal loop or fee math is wrong, alpha users will lose trust immediately.
