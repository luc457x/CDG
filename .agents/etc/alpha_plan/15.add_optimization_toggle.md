# Phase 15: Add portfolio optimization toggle (default ON)
**Priority:** P1 — High value for alpha testing  
**Module:** `src/pipeline.rs`, `src/main.rs` (base_plan P5)

## Goal
Allow alpha users to disable the 10k-simulation Monte Carlo step when they only need price data + indicators.

## Tasks
1. Add `optimize: bool` to `PipelineConfig` (default `true`).
2. In `RunPipeline` CLI:
   - `#[arg(long, default_value_t = true, env = "CDG_OPTIMIZE")] optimize: bool`
   - `#[arg(long = "no-optimize")] no_optimize: bool`
   - Effective = `optimize && !no_optimize`.
3. Gate the optimization block at `pipeline.rs:~426` as `if config.optimize && currency_cols.len() >= 2`.
4. When skipped, print `"Skipping portfolio optimization (disabled via --no-optimize / CDG_OPTIMIZE=false)."`.

## Validation
- `cargo build` clean.
- Manual: `cdg collect --coin btc,eth --no-optimize` (or `CDG_OPTIMIZE=false`) → optimization skipped with message, pipeline otherwise succeeds.

## Why Included
Lightweight alpha runs (single coin, quick test) should not force 10k Monte Carlo sims. Toggle removes friction for alpha testers.
