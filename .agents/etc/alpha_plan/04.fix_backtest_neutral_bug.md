# Phase 04: Fix neutral-signal fee/P&L equity bug in backtest.rs
**Priority:** P0 — Alpha Blocking  
**Module:** `src/backtest.rs` (Phase 1 Finding 1.5 / 6c)

## Goal
Fix the equity loop in `run_backtest_for_asset` so that transitioning from long (2) to neutral (1) charges the exit fee correctly and locks in the previous day's P&L before the fee.

## Tasks
1. Rewrite the equity loop to track an explicit `position: i32` (`-1` short, `0` neutral, `1` long) held at the **start** of the bar.
2. On a transition `prev_position → new_position`:
   - Charge fee on the full equity base: `eq_base *= 1.0 - (fee + slippage)`.
   - Apply the day's return `r_t * conf_t` with the **sign of `prev_position`**, not `new_position`.
   - This means entering neutral from long locks in the long day's return and deducts the exit fee.
3. Remove the ambiguous `current_position` variable; replace with `prev_position`.
4. Add a deterministic test:
   - 3-bar price series: [100, 110, 100]
   - Strategy: RSI < 30 buy, else sell (neutral)
   - Assert exact equity at each bar, including the fee-charged exit bar.

## Validation
- `cargo test -p cdg backtest::` passes.
- New test `test_neutral_exit_fees_pnl` asserts exact equity values.
- Regression: no change to existing Sharpe/drawdown/R2 tests.

## Why This Blocks Alpha
Portfolio backtest returns are wrong by one bar for every neutral exit. Over 90 days this compounds and produces misleading performance metrics.
