# Phase 23: Harden cache.rs internals
**Priority:** P2 — Medium severity API correctness  
**Module:** `src/cache.rs`

## Goal
Close encapsulation and safety gaps in the cache backend.

## Tasks
1. Make `get_internal` and `insert_internal` private (`async fn`, not `pub async fn`) so callers cannot bypass the `CacheBackend` trait.
2. Guard against negative `ttl_secs`: if `ttl_secs < 0`, return `Ok(None)` immediately (negative TTL = "already expired").
3. Update `check_cache_hits` to fetch concurrently using `futures::future::join_all` instead of sequential `.await` loop.
4. Optional: add `max_body_bytes` guard in `insert_internal` (e.g., 10 MB) to prevent disk exhaustion from giant API responses.

## Validation
- `cargo test -p cdg cache::` passes.
- Attempting to call `cache.get_internal(...)` from another module fails to compile (public API audit).
- Negative TTL test: `cache.get(url, -1).await` returns `Ok(None)`.

## Why Included
Public internal methods break trait encapsulation and make refactoring harder. Alpha maintenance depends on a clean API surface.
