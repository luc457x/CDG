# Phase 05: Fix backtest metrics accuracy + remove dead prediction_r2
**Priority:** P0 — Alpha Blocking  
**Module:** `src/backtest.rs` (Phase 1 Findings 1.6 / 6b / 6d)

## Goal
Make `BacktestMetrics` math correct and remove misleading hardcoded zeros.

## Tasks
1. **Fix `calculate_r2`** (`backtest.rs:~279-297`): When `ss_tot == 0.0`, return `1.0` if `ss_res == 0.0` (perfect constant prediction), else `0.0`. Add test for perfect prediction of constant series.
2. **Fix `calculate_max_drawdown`** (`backtest.rs:~318-336`): Change guard from `peak > 0.0` to `peak != 0.0` so negative-peak equity is handled correctly.
3. **Resolve `prediction_r2`** (`backtest.rs` struct + `run_backtest_for_asset` + `backtest_portfolio`):
   - Option A (preferred for alpha): compute it as strategy returns vs buy-and-hold returns using `calculate_r2`.
   - Option B (minimum): remove `prediction_r2` from `BacktestMetrics`, the CSV/JSON report schema, and the table formatter until it is correctly computed. **Do not keep exporting `0.0` unconditionally.**
   - Also drop the placeholder `prediction_accuracy: 1.0`, `active_win_rate: 1.0`, and `prediction_rating: "N/A"` from portfolio/treasury metrics. Replace with `NotApplicable` or remove the fields.

## Validation

  - `test_r2_constant_series_perfect_prediction` → returns `1.0`.
