# Phase 25: Add pipeline currency_cols assertion
**Priority:** P3 — Safety rail  
**Module:** `src/pipeline.rs`

## Goal
Prevent a potential panic in lightweight mode where `currency_cols` could be empty and `currency_cols[0]` would index out of bounds.

## Tasks
1. In `run_pipeline_flow`, after the currency fetch loop, add:
   ```rust
   assert!(!currency_cols.is_empty(), "currency_cols is empty after fetch loop — all currency tasks failed or no coins were supplied");
   ```
2. If the project prefers `Result` over `assert!` in library code, change to:
   ```rust
   if currency_cols.is_empty() {
       return Err(anyhow!("No currency data was fetched — check coin IDs and network connectivity"));
   }
   ```
3. Verify no other `currency_cols[0]` or similar unguarded indexing exists in `pipeline.rs`.

## Related Gap (parallel safety rail)
The parallel safety rail at `main.rs:495` (`currency_dfs[0]` panic when all OHLC fetch tasks fail) is a separate but related gap. Guard it with the same `if is_empty { return Err(...) }` pattern in the `run_ohlcv_flow` path.

## Validation
- `cargo build` clean.
- Negative path: simulate all-coins-failing (mock or manual) → clean error message, not panic.

## Why Included
One-line safety rail. Prevents an obscure panic that would otherwise bite an alpha user with a bad coin ID.
