# Phase 26: Fix optimization covariance + Monte Carlo hardening
**Priority:** P1 — Must fix before wider alpha distribution (Critical severity)
**Module:** `src/optimization.rs`
**Source:** `code_review.md` #2 + `alpha_readiness_plan_2026-07-07.md` PHASE 6

## Goal
Fix the near-singular covariance annualization that degenerates Markowitz weights to zero, and close the remaining optimization-hardening gaps (sim cap, explicit risk-free rate, xorshift min-weight hack).

## Tasks
1. **Fix covariance annualization** (`optimization.rs:~112`): `cov_matrix[i][j] = daily_cov * (factors[i] * factors[j]).sqrt()` is inconsistent when factors differ (365 vs 252) and can produce a near-singular / poorly-conditioned matrix → zero weights.
    - Replace with a single unified annualization factor `ann` for the whole portfolio, or use the finance-standard `daily_cov * min(fi, fj)` off-diagonal scaling. Document the chosen convention.
    - Add a regression test: 2-asset portfolio with one crypto (365) + one stock (252) → both weights strictly `> 0.0`.
2. **Add `max_simulations` cap** (default 10,000, hard cap 50,000) to `run_monte_carlo`. Document per-sim memory (~24 B × 10k = 240 KB).
3. **Make risk-free rate an explicit parameter** instead of silently falling back to `0.0` when `^TNX` is absent. Default to `^TNX`; if missing, use a documented constant (e.g., `0.04`) and `eprintln!` a warning. Do not silently compute a returns-only ratio.
4. **Replace the `+ 0.01` min-weight hack** (lines ~201-208) with documented Dirichlet sampling or reject-and-resample so the minimum weight is actually guaranteed; OR document the bias explicitly if kept.
5. **Fix progress bar** under Rayon: update `pb` from inside the parallel loop (or via a channel) so it reflects real-time progress instead of jumping 0→100%.
6. **Add formula tests**: weights sum to 1.0; min-vol portfolio volatility `<=` max-sharpe portfolio volatility; Sharpe computed as `(mean - rf) / std * ann.sqrt()`.

## Validation
- `cargo test -p cdg optimization::` passes.
- New test `test_covariance_no_zero_weights` → both assets get `> 0.0` weight with mixed annualization factors.
- `max_simulations` cap is respected (assert loop count).
- Determinism preserved with `--seed`.

## Why This Blocks Wider Alpha
`code_review.md` #2 rates this **Critical**: the current formula yields degenerate zero-weight portfolios, making the optimization output useless for crypto+stock mixes. This is the only Critical-severity finding not otherwise covered by phases 01–25.
